Variables

Variables

Variables are alphanumeric and can contain the underscore character (_). It may not start with a number or capital letter. The values can be reassigned later on in the code.

Constants

Variables that begin with a capital letter are constants and cannot be changed

Local Variables

Local variables can only be accessed within their respective code blocks. They cannot be accessed or changed outside of them.

Global Variables

Global variables are accessible from anywhere in the program. To create a global variable a dollar sign ($) is added to the beginning of the variable.

$global_variable

Arrays

Arrays

Arrays are referenced starting from index 0. To reference from the end of the array, start from -1.

Creating an Array

variable = [element, element, "element"]

Push insert elements at the end.
The << operator inserts a single element at the end.
The insert method inserts multiple elements at the specified index.

Push array.push(element)
<< array << element
Insert array.insert(index,element, element)

Removing Elements

Pop removes the last element.
The delete_at method deletes the element at that index.
Both methods return the elements.

Pop array.pop
Delete At array.delete_at(index)

Printing Arrays

Puts outputs a single element per line.
Print outputs the elements separated by commas on a single line enclosed in brackets.

Combining Arrays

Two arrays can be added to one another to combine them.

When two arrays are subtracted, elements that occur in the second array are removed from the first.

Duplicating Elements

Arrays can be multiplied by an integer to repeat the elements of the array at the end

Hashes

Hashes

Hashes are similar to arrays, where elements can be indexed with strings, numbers or arrays. They come in key=>value pairs. Symbols are keys that are fixed and cannot be changed.

hash_a = {"string"=>value, :symbol=>value}
hash_b = {number=>value; [element, element]=>value}
hash_c = {symbol:value}
puts hash[:symbol]

Nested Hashes

Similar to arrays, hashes can also have multiple levels of nesting.

hash = {
  symbol_a: {key_a:value_a,key_b:value_b}
  symbol_b: {key_c:value_c,key_d:value_d}
}
puts hash[:key][:key]

Other Methods

Returns an Array of Keys hash.keys

Return Values

Classes

Classes

A class defines the data and actions associated with an object. The initialize method is automatically called when a new object is instantiated. Instance variables defined in a class have the @ symbol in the front. Methods can also be defined int he class to accept arguments and return values.

class Class_name
  def initialize
    @variable = value
  end

  def variable #Getter
    @variable
  end

  def variable=(parameter) #Setter
    @variable = parameter
  end
end

object = Class_name.new

Accessors

While you can manually create methods that will perform the action of setting and getting instance variables. There is an easy way of creating these methods. The self keyword is used within the class to reference the instance variables and can be used to differentiate from methods with the same name.

attr_reader :variable #Getter Only

attr_writer :variable #Setter Only

    #Statements

Variables with two @ signs are variables that belong to the class. They are shared amoung created objects and tell you information about the class.

Overridding

All classes come with a to_s method that converts the object into a string. When the to_s method is used on an object, it will print the object id. However, the to_s method can be overridden to return a string can be used to print information about the object.

class Class_name
   def to_s
    "String"
  end
end

Omitting the to_s will produce the same result as including it, the to_s is automatically called for objects.

print object.to_s
print object

Inheritance

Subclasses (derived) inherit the attribute and behaviors of its superclass (base). Methods can be overridden to have its own implementation. There can be multiple levels of inheritance with an "is a" relationship of base and derived class.

class Subclass < Superclass
  #Statements
end

Super Method

When the super method is used in a derived classes method, the base classes method of the same name will be called.

class Subclass < Superclass
  def method_name
    super
    #Statements
  end
end

The super method can be used to initialize the instance variables.

class Subclass < Superclass
  def initialize(parameter, parameter)
    super(parameter)
    @variable = value
  end
end
